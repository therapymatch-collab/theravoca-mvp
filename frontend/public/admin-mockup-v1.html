Testing tools -- hidden in production

Everything in this section only renders when the app is running in staging or dev mode. In production, this primary tab is hidden so admins can't accidentally wipe live data.

Test actions

Scraper test
Live data refresh
Refresh all caches

Bust the matching-defaults cache, snapshot cache, capacity cache, and Outcomes dashboard cache so the next page load reads fresh from Mongo.

Seed data
Backfill with test data

Drop a known set of test patient requests + matching applications + feedback so the Outcomes dashboard has something to render. Safe to re-run -- idempotent.

Destructive
Wipe test data

Delete every request, application, feedback, and therapist_survey whose email contains "+test" or matches the test allowlist. Real patient data is untouched. Confirmation modal required.

Cron triggers
Run a cron now

Manually fire any scheduled job without waiting for the next tick. Useful when verifying a fix shipped to staging.

Email preview
Send test email to me

Render any template against test data and send it to your inbox so you can see what patients/therapists actually receive.

v1 cleanup
Strip legacy flags

One-shot Mongo cleanup that removes legacy structured_followup_*_sent_at flags from old requests. Idempotent.

Matching simulator

Run a hypothetical patient request and see who would match.

Research reviews

Inspect deep-research output for any therapist (DuckDuckGo + 5 web pages + AI extraction). Compare cached vs. fresh. Re-run on demand.

Feedback test tools

Fire individual surveys against a request, reset survey flags, manually trigger Match Strength recompute, toggle global testing mode.

Scraper test

Run the directory scraper against a single URL and inspect output before bulk import.
