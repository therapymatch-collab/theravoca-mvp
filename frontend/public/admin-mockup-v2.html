Recruiting · Track A

Per-request outreach

Auto-fires when an incoming patient request gets fewer than the target number of matches. Outreach is scoped to that specific request.

▶ How Track A works
click to expand
When a patient submits intake and the matcher returns fewer than min_results therapists above the threshold, a recruiting run fires for that request. Sources cascade: Google Places → Psychology Today → admin-registered directories → backup HTML scrapers. Email enricher pulls real contacts off the therapist's website. Email-ready candidates get a templated invite that mentions the patient's specific brief; SMS-ready get the SMS fallback. The run keeps state on the request itself, so a later patient with the same need won't re-poke the same therapists for 14 days.

Recruiting · Track B

General gap-fill

Proactive outreach driven by the Coverage gaps report. Currently dry-run — creates drafts only, sends nothing.

▶ How Track B works
click to expand
A nightly cron reads the Coverage gaps report. For each critical/warning gap, it builds a target list of therapists with that specialty in cities where you have demand, runs the same enricher pipeline as Track A, and produces draft invites. While dry-run is on, drafts pile up in this view but no email goes out. "Flip to live" sets dry_run=False in the database (no code deploy needed) and the next nightly run will send.

Matching pipeline

Default threshold: 80% · Cap: 97 · Top-N sent as matches changed from 70%

▶How matching works
click to expand
Patient request flows through hard filters first (must pass all to be considered), then each remaining therapist gets a weighted score across multiple soft axes, plus bonuses for deeper signals. The final score is mapped to a 0-97 display range; above the patient's threshold (default 80%) we send the top N as matches. Cap of 97 leaves headroom for future weight tuning without re-norming.
